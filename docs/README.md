DocAgent cross-repo test — source code

## API

### `get_user(user_id, include_email=False)`

Возвращает пользователя по ID.

**Параметры:**
- `user_id` (int) — уникальный идентификатор пользователя.
- `include_email` (bool, по умолчанию `False`) — включить email в ответ.

**Возвращает:** словарь с полем `id`. Если `include_email=True`, также содержит поле `email`.

---

### `create_user(name, email, role="user")`

Создаёт нового пользователя.

**Параметры:**
- `name` (str) — полное имя пользователя.
- `email` (str) — адрес электронной почты (должен быть уникальным).
- `role` (str, по умолчанию `"user"`) — роль пользователя.

**Возвращает:** словарь с полями `name`, `email`, `role`.

---

### `delete_user(user_id)`

Удаляет пользователя безвозвратно. Операция необратима.

**Параметры:**
- `user_id` (int) — уникальный идентификатор пользователя.

**Возвращает:** `bool`.

---

## Класс `UserService`

Сервис для управления пользователями.

### `__init__(db, cache=None)`

**Параметры:**
- `db` — объект базы данных.
- `cache` (опционально) — объект кэша.

### `find_by_email(email)`

Находит пользователя по адресу электронной почты.

**Параметры:**
- `email` (str) — адрес электронной почты.

**Возвращает:** результат запроса к базе данных.
